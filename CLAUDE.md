## Project

Two implementations of a Pomodoro timer:

| File | Type | Theme | Design |
|------|------|-------|--------|
| `pomodoro_app.py` | Python tkinter desktop | Brandy (warm terracotta/cream) | Square box, Canvas rectangles |
| `pomodoro-timer.html` | HTML single-file web app | Dark (#1a1a2e) | Circular SVG ring |

## Running

```bash
# Python desktop app
python3 pomodoro_app.py

# macOS app bundle (double-clickable, no Terminal)
open Pomodoro.app

# HTML — open in any browser
open pomodoro-timer.html
```

## Architecture — `pomodoro_app.py`

- Pure Python 3 + tkinter, zero dependencies beyond stdlib
- Single class `PomodoroApp` in one file (~470 lines)
- macOS-specific: uses `osascript` for sound (`beep`) and notifications (`display notification`), `-topmost` window attribute
- All UI drawn on a single `tk.Canvas` (no widgets like Button/Label) — uses `create_rectangle` and `create_text` with tag-based event bindings (`tag_bind`)
- Font tuples are single-entry on macOS: `("Menlo", 36, "bold")` — multi-font tuples cause `TclError`
- Three modes: work (25 min), break (5 min), long break (15 min), auto-switch on finish (every 4th pomodoro → long break)
- No persistent stats (stats reset on app restart), unlike the HTML version which uses `localStorage`

# CVC 投后管理看板 (Post-Investment Management Dashboard)

## Overview

Single-file HTML dashboard for a Corporate Venture Capital (CVC) post-investment management workflow. Built for a CVC team managing 15 A-share listed portfolio companies under a simulated DJI (大疆创新) CVC context.

**File:** `post-investment-dashboard.html` (~1,416 lines)

**Subject:** CVC portfolio monitoring and actionable recommendations  
**Audience:** Investment managers, risk committee, leadership  
**Design Language:** Bloomberg Terminal-inspired dark theme with amber accents

## Running

```bash
# Open directly in browser
open post-investment-dashboard.html

# Or serve via HTTP (needed for some features)
python3 -m http.server 8765
open http://localhost:8765/post-investment-dashboard.html
```

## Architecture

- **Pure single-file HTML** — zero build step, no framework, no backend
- **CSS:** CSS custom properties (design tokens), Grid + Flexbox layout, 3 responsive breakpoints (1200/768/520px)
- **Charts:** Chart.js 4.4.0 via CDN — bar, line, doughnut, radar, bubble, and mixed chart types
- **JS:** ~950 lines in a single IIFE (`(function(){...})()`), organized by data → aggregation → safe-rendering → events. Each section render is wrapped in try-catch via `safeRender()`.
- **Data:** Mock data generated client-side via `generateMockData()` — 15 companies × 100+ fields each. Financial metrics sourced from real 2024 annual reports (revenue, net profit, R&D intensity, market position).
- **State:** `portfolioData[]`, `fundAgg{}`, `charts{}` instance registry, `recommendations{capital,noncapital}`, `selectedCompanyId`, `activeRole`, `activeRecType`
- **Risk model:** Five-dimension scores *derived from* individual risk factors (not the reverse). Base 95 per dimension, minus severity-weighted deductions for active factors.
- **Navigation:** Sticky top navbar with IntersectionObserver scroll-spy + inline `onclick` fallbacks on all critical buttons
- **Global API:** `window.AppNav.refresh()`, `window.AppNav.scrollToSection(id)`, `window._renderFilteredFactors()`

## Five-Dimension Risk Framework

The risk model uses a weighted 5-dimension scoring system (0–100, lower = riskier):

| Dimension | Weight | Sub-Indicators | Panel Color |
|-----------|--------|----------------|-------------|
| 外部风险 (External) | 15% | 政治环境 · 行业变化 · 友商竞争 · 法律法规 · 贸易政策 | Red |
| 运营风险 (Operational) | 20% | 质量管理 · 团队稳定性 · 供应链风险 · 技术产品 | Amber |
| 财务风险 (Financial) | 30% | 资金流动性 · 运营效率 · 财务结构 · 造血能力 | Blue |
| 战略风险 (Strategic) | 15% | 业务组合策略 · 技术变革 · 战略转型 · 市场分析 | Purple |
| 治理风险 (Governance) | 20% | 治理结构 · 关键股东控制 · 关联交易 · 内控合规 | Teal |

Traffic light mapping: ≥75 Green / 60–74 Yellow / <60 Red.

Each company also generates `riskFactors[]` — specific risk events with dimension, subCategory, severity (high/medium/low), status (active/monitoring/mitigated), owner, and deadline.

## 2D Quadrant Classification (商业成熟度 × 战略重要性)

Replaces traditional ABCDE grading with a two-dimensional matrix:

**Business Maturity** (商业成熟度, 5 sub-dimensions):
- 创始人及团队 · 战略及商业逻辑 · 价值创造 · 管理运营 · 资本运营

**Strategic Importance** (战略重要性, 2 sub-dimensions):
- 对我司业务重要程度 · 营收规模及产业链价值

| Quadrant | Maturity | Importance | Color | Strategy |
|----------|----------|------------|-------|----------|
| ⭐ 明星 (Star) | ≥65 | ≥65 | Green | 核心持有+择机增资 |
| 🏛 基石 (Cornerstone) | ≥65 | <65 | Blue | 持有为主+财务退出 |
| 🌱 培育 (Cultivate) | <65 | ≥65 | Amber | 战略培育+业务赋能 |
| 👀 观察 (Observe) | <65 | <65 | Red | 观察跟踪+择机退出 |

Visualized as a full-width horizontal bubble chart (X = maturity, Y = importance, bubble size = valuation).

## Key Design Decisions

1. **Single file, no build step** — can be opened directly from Finder or served via any HTTP server
2. **Chart lifecycle management** — all charts destroyed and recreated on refresh to prevent memory leaks
3. **Inline onclick fallbacks** — critical buttons (refresh, nav links) use both JS event listeners AND inline `onclick` attributes so the page is functional even if `bindEvents()` fails
4. **Safe render wrapper** — `renderAll()` wraps each section render in try-catch and collects errors rather than crashing
5. **Mock data determinism** — uses index-based assignment (not purely random) for consistent demo appearance across refreshes
6. **Cache-disabled** — meta tags prevent browser caching during active development
7. **Bloomberg aesthetic** — amber-on-dark, tabular figures, no gradients, no large shadows, scan-line header effect
8. **CVC differentiation** — BU engagement heatmap and strategic importance axis distinguish this from a generic VC dashboard
9. **Risk score derived from factors** — dimension scores (0–100) are calculated from individual risk factors, not the reverse. Each active factor deducts points from a base of 95. The dimension table and risk factors table are traceably linked.
10. **Filterable risk factors** — the risk factors table has 4 dropdown filters (company, dimension, severity, status) operating with AND logic. This is the practical entry point for drill-down analysis.

- Exit DPI projection is mock; needs actual LP distribution schedules
- No persistent storage — data regenerates on every refresh
- No export capability (PDF/Excel for IC meetings)
